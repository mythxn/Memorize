Game Report - Memorize - CSD2500 - Mithun Shanoj - M00739358

A basic game that the user can play and obtain a score.

All game code is saved inside game.js, with the main method ranging from lines 9-24, and
other additional sub-methods in the same file.


Advanced JavaScript such as objects and arrays.

Arrays and objects are used to store data depending on the context and type of data being stored.

  • Arrays: Used to store words that are displayed to the user in the game.
  • Objects: Used to save user data into session and local storage.


Advanced game features, such as graphics, animation, multiple skill levels.

Memorize has two different levels and animations are integrated to make the game feel more alive.

  1. Graphics: Used across pages that all follow a common colour scheme to make the user interface more fun to navigate thorough.

    img tags were used to show these illustrations.


  2. Animation: Used on the game page to add to the game experience which also matches the typing theme of the game.
  3. Levels: There are two levels, easy and hard. Easy gives the user more time to memorise the word while hard gives lesser time.

User registration and login with JavaScript validation and error message.


Storage of users scores in JSON format using HTML local storage


Sort by the score from high to low.

All sorting related code is stored in leaderboard.php and sorttable.js


List of 3rd-party library used and the feature used.
